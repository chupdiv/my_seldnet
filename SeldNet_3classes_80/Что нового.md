### Обновление генератора датасета

Обновлён скрипт `research/Localization_neural/generate_reverb_all_inclusive/generate_reverb_all_inclusive_gpu_voice.py`.

Что добавлено и исправлено:

1. **Параметризация под обучение**
   - Добавлены CLI-параметры:
     - `--fs` — целевая частота дискретизации;
     - `--n-mics` — число микрофонов (поддерживаются геометрии на 4 и 5 каналов);
     - `--seed` — seed генерации;
     - `--output-dir` — выходной каталог датасета.
   - Имя выходного каталога по умолчанию теперь включает `fs` и `n_mics`, чтобы не смешивать разные версии датасета.

2. **Исправление логики классов**
   - Исправлена ошибка в сценарии `both`: вероятность `VOICE_PROB` теперь интерпретируется так же, как в `env_only` (при `r <= VOICE_PROB` выбирается голос).

3. **Проверка качества сгенерированного датасета**
   - Добавлен флаг `--run-sanity-check` для пост-проверки результата:
     - парность `wav/csv`;
     - соответствие `fs` и числа каналов ожидаемым параметрам;
     - формат и диапазоны метаданных (`frame`, `azimuth`, `elevation`, `distance`);
     - сводка по классам и количеству файлов.

Пример запуска:

`python generate_reverb_all_inclusive_gpu_voice.py --fs 250000 --n-mics 4 --run-sanity-check`

4. **Смесь «дрон + окружение» и уровни**
   - По умолчанию **без** пик-нормализации каждого источника в `render_source` (сохраняется энергетика для SNR); при перегрузе — только масштабирование до безопасного пика (`FLOAT_WAV_PEAK_TARGET`). Включить старое поведение: `--peak-normalize-render`.
   - SNR между дроном и фоном задаётся случайно в диапазоне **[-6, 6] дБ** (по умолчанию); мощность фона считается по **активным интервалам** событий, один коэффициент на все каналы. Параметры: `--snr-min`, `--snr-max`.
   - **Train/test по дронам**: `fold3` использует только файлы из `DroneAudioData`, `fold4` — только из `DroneAudioDataTest` (если соответствующая папка не пуста), чтобы снизить утечку исходников.
   - Отбор `good_env_files`: по умолчанию **смежные** интервалы не считаются пересечением; старое правило: `--overlap-touch-counts`.
