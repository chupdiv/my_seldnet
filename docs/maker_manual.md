## 1. Общие сведения

`maker.py` — это скрипт-оркестратор, который автоматизирует полный цикл подготовки данных и обучения модели SELD (Sound Event Localization and Detection). Скрипт выполняет Три последовательных этапа, каждый из которых запускается **только при необходимости** — если изменились параметры, обновился датасет или отсутствуют требуемые файлы. Если ничего не изменилось, этапы генерации и расчёта признаков пропускаются, а выполнение переходит сразу к обучению.

**Структура проекта:**

```
project/
├── maker.py                          ← этот скрипт
├── generate_gpuRIR_3c.py             ← генератор отражений
├── SeldNet_3classes_80/
│   ├── parametres.py                 ← параметры задачи
│   ├── batch_feature_extraction.py   ← расчёт признаков
│   └── train_seldnet.py             ← обучение модели
├── SeldNet_3classes_200/
│   ├── parametres.py
│   ├── batch_feature_extraction.py
│   └── train_seldnet.py
└── .maker_state/                     ← кэш состояния (создаётся автоматически)
    ├── SeldNet_3classes_80_6.json
    └── SeldNet_3classes_200_4.json
```

## 2. Краткое описание этапов

| Этап | Описание                               | Скрипт                          | Запускается всегда?                                                                    |
| -------- | ---------------------------------------------- | ------------------------------------- | ------------------------------------------------------------------------------------------------------- |
| 1        | Генерация отражений (gpuRIR) | `generate_gpuRIR_3c.py`             | Нет — при изменении параметров или отсутствии данных       |
| 2        | Расчёт признаков                | `<dir>/batch_feature_extraction.py` | Нет — при изменении параметров или отсутствии `feat_label_dir` |
| 3        | Обучение нейросети            | `<dir>/train_seldnet.py`            | Да (если не указан `--skip-train`)                                                      |

**Ключевая логика:**

- Этап 1 проверяет набор параметров `STAGE2_PARAM_KEYS` и наличие каталогов `metadata_dev/` и `mic_dev/` внутри `dataset_dir`.
- Этап 2 запускается, если был выполнен этап 1, либо изменились параметры из `STAGE3_PARAM_KEYS`, либо отсутствует `feat_label_dir`.
- Этап 3 не имеет условий пропуска — он запускается при каждом вызове, если не указан флаг `--skip-train`.

## 3. Аргументы командной строки

```
python maker.py --dir <каталог> --task <задача> [--log <файл>] [--skip-rir] [--skip-feats] [--skip-train]
```

### Обязательные аргументы

| Аргумент | Описание                                                                                                                                                                                                                                                     |
| ---------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `--dir`        | Конфигурационный каталог. Допустимые значения:`SeldNet_3classes_80`, `SeldNet_3classes_200`. Каталог должен содержать `parametres.py`, `batch_feature_extraction.py` и `train_seldnet.py`. |
| `--task`       | Идентификатор задачи (целое число или строка). Передаётся в функцию `get_params(task_id)` из файла `parametres.py`.                                                                               |

### Опциональные аргументы

| Аргумент                    | Описание                                                                                                                                                                                                                                                                                                            |
| ----------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `--log <файл>`                | Путь к файлу журнала. Весь вывод (включая вывод дочерних процессов) дублируется в этот файл в режиме добавления. Если директория не существует, она создаётся автоматически. |
| `--count <целое число>` | Объем выборки записей с отражениями, формируемых на 1 этапе                                                                                                                                                                                                                |
| `--skip-rir`                      | Пропустить этап генерации отражений. Полезно, когда датасет уже существует и менять его не нужно.                                                                                                                                          |
| `--skip-feats`                    | Пропустить этап расчёта признаков. Полезно, когда признаки уже рассчитаны и нужно только переобучить модель.                                                                                                                    |
| `--skip-train`                    | Пропустить этап обучения нейросети. Полезно, когда нужно только подготовить данные (генерация + признаки).                                                                                                                          |

## 4. Хранение состояния между запусками

Скрипт хранит информацию о предыдущих запусках в каталоге `.maker_state/` (расположен в рабочей директории, создаётся автоматически).

### Именование файлов состояния

Каждая комбинация `--dir` и `--task` получает собственный файл:

```
.maker_state/SeldNet_3classes_80_6.json    ← dir=SeldNet_3classes_80, task=6
.maker_state/SeldNet_3classes_200_4.json   ← dir=SeldNet_3classes_200, task=4
```

Формат имени: `{dir_name}_{task_id}.json`

Это означает, что **разные комбинации dir/task независимы друг от друга**. Запуск с `--dir SeldNet_3classes_80 --task 6` и с `--dir SeldNet_3classes_80 --task 7` используют разные файлы состояния и не влияют друг на друга.

### Что хранится в состоянии

- **params** — полная копия словаря параметров, возвращённого `get_params(task_id)`.
- **dataset_hash** — MD5-хеш содержимого `dataset_dir` (пути файлов, размеры, время модификации).
- **last_run** — временная метка последнего запуска в формате ISO 8601.
- **stage2_executed** — был ли выполнен этап 1 при последнем запуске.
- **stage3_executed** — был ли выполнен этап 2 при последнем запуске.

## 5. Проверка параметров

Этот этап выполняется всегда и не может быть пропущен.

**Что происходит:**

1. Скрипт динамически импортирует `<dir>/parametres.py` и вызывает `get_params(task_id)`.
2. Загружается файл состояния из `.maker_state/`.
3. Текущие параметры сравниваются с сохранёнными. Изменения выводятся в консоль в формате «было / стало».
4. Определяется, какие параметры изменились, и классифицируются по этапам:
   - Изменения в `STAGE2_PARAM_KEYS` → пометка «нужен этап 1».
   - Изменения в `STAGE3_PARAM_KEYS` → пометка «нужен этап 2».
5. Вычисляется хеш содержимого `dataset_dir` и сравнивается с предыдущим.

**Пример вывода при первом запуске:**

```
ЭТАП 1: Загрузка параметров и проверка изменений
============================================================

Загрузка модуля: SeldNet_3classes_80/parametres.py
Параметры получены (task_id='6').

Предыдущий запуск не найден — первое выполнение.
Все текущие параметры (первый запуск):
  fs = 16000
  n_mics = 4
  label_hop_len_s = 0.1
  ...
```

**Пример вывода при изменении параметров:**

```
Сравнение параметров с предыдущим запуском:
    Найдено изменённых параметров: 2
      n_mics:
        было:  2
        стало: 4
      seed:
        было:  20260405
        стало: 12345

Изменились параметры этапа 2 (генерация отражений): ['n_mics', 'seed']
```

### Параметры, отслеживаемые для этапа 2

Следующие ключи из `params` влияют на решение о перегенерации отражений:

| Ключ            | Описание                                                              |
| ------------------- | ----------------------------------------------------------------------------- |
| `fs`              | Частота дискретизации                                     |
| `n_mics`          | Количество микрофонов                                     |
| `label_hop_len_s` | Шаг кадров меток (секунды)                               |
| `seed`            | Зерно генератора случайных чисел                 |
| `max_audio_len_s` | Максимальная длительность аудио (секунды) |
| `lowpass_hz`      | Частота среза ФНЧ (по умолчанию 0)                  |

### Параметры, отслеживаемые для этапа 3

Следующие ключи из `params` влияют на решение о перерасчёте признаков:

| Ключ                   | Описание                                          |
| -------------------------- | --------------------------------------------------------- |
| `hop_len_s`              | Шаг STFT (секунды)                              |
| `nb_mel_bins`            | Количество мел-полос                    |
| `mel_fmin_hz`            | Нижняя частота мел-шкалы             |
| `mel_fmax_hz`            | Верхняя частота мел-шкалы           |
| `use_salsalite`          | Флаг использования SALSA-Lite            |
| `raw_chunks`             | Размер сырых чанков                      |
| `saved_chunks`           | Размер сохраняемых чанков          |
| `fmin_doa_salsalite`     | Нижняя частота DOA (SALSA-Lite)              |
| `fmax_doa_salsalite`     | Верхняя частота DOA (SALSA-Lite)            |
| `fmax_spectra_salsalite` | Верхняя частота спектра (SALSA-Lite) |

> **Важно:** Изменение параметра, которого нет ни в одном из этих списков (например, гиперпараметры обучения), **не вызовет** перегенерацию отражений или перерасчёт признаков.

## 6. Этап 1 — Генерация отражений (gpuRIR)

### Условия выполнения

Этап 1 запускается, если выполняется **хотя бы одно** из условий:

1. Изменился хотя бы один параметр из `STAGE2_PARAM_KEYS`.
2. Отсутствует `dataset_dir` (каталог, указанный в `params["dataset_dir"]`).
3. Отсутствует подкаталог `metadata_dev/` внутри `dataset_dir`.
4. Отсутствует подкаталог `mic_dev/` внутри `dataset_dir`.

### Архивирование

Если этап 1 запускается и `dataset_dir` существует, скрипт **автоматически архивирует** его, добавляя числовой суффикс (подробнее см. [раздел 9](#9-архивирование-каталогов)).

### Команда, выполняемая на этапе 1

```bash
python generate_gpuRIR_3c.py \
    --fs <fs> \
    --n-mics <n_mics> \
    --frame-step <label_hop_len_s> \
    --run-sanity-check \
    --lowpass-hz <lowpass_hz> \
    --seed <seed> \
    --duration <max_audio_len_s>
```

### Пример: пропуск этапа 1

Если датасет уже сгенерирован и его нужно сохранить:

```bash
python maker.py --dir SeldNet_3classes_80 --task 6 --skip-rir
```

Скрипт выведет:

```
ЭТАП 1: Генерация отражений (gpuRIR)
======================================================================
Пропуск: генерация отражений отключена флагом --skip-rir.
```

## 7. Этап 2 — Расчёт признаков

### Условия выполнения

Этап 2 запускается, если выполняется **хотя бы одно** из условий:

1. Этап 2 был выполнен в текущем запуске (свежий датасет → нужны новые признаки).
2. Изменился хотя бы один параметр из `STAGE3_PARAM_KEYS`.
3. Отсутствует `feat_label_dir` (каталог, указанный в `params["feat_label_dir"]`).

### Команда, выполняемая на этапе 2

```bash
python <dir>/batch_feature_extraction.py <task_id>
```

Скрипт `batch_feature_extraction.py` внутри себя вызывает `get_params(task_id)`, поэтому дополнительная передача параметров не требуется.

### Каскадный перезапуск

Если изменились параметры этапа 1, то запускается этап 1 и затем автоматически запускается этап 2 (новые данные → новые признаки). Флаг `--skip-feats` подавляет этот каскад.

> **Внимание:** Если вы указали `--skip-feats` и при этом изменились параметры этапа 1 (была запущена генерация отражений), признаки **не будут пересчитаны автоматически**. Вам придётся либо запустить скрипт повторно без `--skip-feats`, либо вручную запустить `batch_feature_extraction.py`.

---

## 8. Этап 3 — Обучение нейросети

### Условия выполнения

Этап 3 запускается **всегда**, если не указан флаг `--skip-train`. Никакие параметры или изменения датасета не влияют на решение о запуске — обучение выполняется при каждом вызове.

### Имя модели

Имя модели формируется автоматически как:

```
<YYYYMMDD>
```

Например, при `--task 6` и дате 19.04.2026 имя будет: `20260419`.

**ВНИМАНИЕ!** Если вы запустите скрипт дважды в один день с одним `task_id`, имя модели будет **одинаковым** — предыдущие файлы модели могут быть перезаписаны.

### Команда, выполняемая на этапе 3

```bash
python <dir>/train_seldnet.py <task_id> <task_id>-<YYYYMMDD>
```

## 9. Архивирование каталогов

Когда скрипт принимает решение перегенерировать отражения (этап 1), а каталог `dataset_dir` уже существует, он не удаляет его, а **переименовывает**, добавляя числовой суффикс.

### Алгоритм архивирования

1. Скрипт проверяет, существует ли `dataset_dir`.
2. Если существует — ищет наименьший свободный числовой суффикс.
3. Переименовывает каталог и создаёт новый.

### Примеры последовательных запусков

```
Запуск 1:  dataset_dir не существует → генерация → dataset_dir/ создан
Запуск 2:  параметры не изменились → пропуск этапа 2
Запуск 3:  изменился seed → архивация → dataset_dir.1/ → генерация → dataset_dir/ создан
Запуск 4:  изменился n_mics → архивация → dataset_dir.2/ → генерация → dataset_dir/ создан
```

### Особенности

- Суффиксы нарастают последовательно: `.1`, `.2`, `.3` и т.д.
- Скрипт проверяет наличие **и каталога без суффикса, и всех суффиксов** при поиске свободного номера. То есть если существуют `dataset_dir/` и `dataset_dir.1/`, следующий будет `dataset_dir.2/`.
- Архивация применяется **только к `dataset_dir`**, каталог `feat_label_dir` не архивируется.
- Архивация происходит **до** запуска генерации. Если генерация завершится с ошибкой, старый каталог уже переименован — его можно вернуть вручную.

## 10. Примеры использования

### 10.1. Первый запуск (базовый сценарий)

```bash
python maker.py --dir SeldNet_3classes_80 --task 6
```

**Что произойдёт:**

1. Проверка параметров: параметры загружены, предыдущего запуска нет — все параметры выведены как «новые». Датасета нет.
2. Этап 1: `dataset_dir` отсутствует → запускается генерация отражений.
3. Этап 2: этап 1 был выполнен → запускается расчёт признаков.
4. Этап 3: запускается обучение. Имя модели: `6-20260419`.
5. Состояние сохранено в `.maker_state/SeldNet_3classes_80_6.json`.

### 10.2. Повторный запуск без изменений

```bash
python maker.py --dir SeldNet_3classes_80 --task 6
```

**Что произойдёт:**

1. Проверка параметров: параметры загружены, сравнение показывает «Изменений параметров нет». Датасет не изменился.
2. Этап 1: параметры не изменились, `dataset_dir` существует, `metadata_dev/` и `mic_dev/` на месте → **пропуск**.
3. Этап 2: этап 1 не выполнялся, параметры признаков не изменились, `feat_label_dir` существует → **пропуск**.
4. Этап 3: запускается обучение (всегда).

### 10.3. Изменение параметров генерации

В `parametres.py` изменён `seed` с `20260405` на `42`:

```bash
python maker.py --dir SeldNet_3classes_80 --task 6
```

**Что произойдёт:**

1. Проверка параметров: обнаружено изменение `seed` (относится к `STAGE2_PARAM_KEYS`).
2. Этап 1: параметры изменились → существующий `dataset_dir/` архивируется в `dataset_dir.1/` → запускается генерация → новый `dataset_dir/`.
3. Этап 2: этап 1 выполнен → расчёт признаков.
4. Этап 3: обучение.

### 10.4. Изменение только параметров признаков

В `parametres.py` изменён `nb_mel_bins` с `64` на `128`:

```bash
python maker.py --dir SeldNet_3classes_80 --task 6
```

**Что произойдёт:**

1. Проверка параметров: обнаружено изменение `nb_mel_bins` (относится к `STAGE3_PARAM_KEYS`).
2. Этап 1: параметры этапа 2 не изменились, датасет на месте → **пропуск**.
3. Этап 2: параметры признаков изменились → перерасчёт признаков.
4. Этап 3: обучение.

### 10.5. Быстрое дообучение (только обучение)

```bash
python maker.py --dir SeldNet_3classes_80 --task 6 --skip-rir --skip-feats
```

**Что произойдёт:**

1. Проверка параметров: обычная.
2. Этап 1: **пропуск** (флаг).
3. Этап 2: **пропуск** (флаг).
4. Этап 3: обучение.

Полезно для быстрой итерации гиперпараметров обучения, когда данные готовы.

### 10.6. Только подготовка данных без обучения

```bash
python maker.py --dir SeldNet_3classes_80 --task 6 --skip-train
```

**Что произойдёт:**

1. Проверка параметров: обычная.
2. Этап 1: генерация (при необходимости).
3. Этап 2: расчёт признаков (при необходимости).
4. Этап 3: **пропуск**.

### 10.7. Логирование в файл

```bash
python maker.py --dir SeldNet_3classes_80 --task 6 --log /var/log/seld/train.log
```

- Файл создаётся в режиме добавления (`append`), предыдущие записи сохраняются.
- Каждый запуск отделяется заголовком с датой и временем.
- Если директория `/var/log/seld/` не существует, она будет создана автоматически.
- Весь вывод (включая дочерние процессы) дублируется и в консоль, и в файл.

### 10.8. Логирование на текущий день

```bash
python maker.py --dir SeldNet_3classes_80 --task 6 --log logs/train-$(date +%Y%m%d).log
```

Создаёт отдельный файл лога на каждый день: `logs/train-20260419.log`.

### 10.9. Работа с другой конфигурацией

```bash
python maker.py --dir SeldNet_3classes_200 --task 4 --log output_200.log
```

Используется отдельный файл `parametres.py` из `SeldNet_3classes_200/` и отдельное состояние в `.maker_state/SeldNet_3classes_200_4.json`.

## 11. Особые случаи и граничные сценарии

### 11.1. Файл параметров не найден

Если указанный в `--dir` каталог не содержит `parametres.py`:

```
[ОШИБКА] Файл параметров не найден: SeldNet_3classes_80/parametres.py
Убедитесь, что каталог 'SeldNet_3classes_80' содержит файл parametres.py.
```

Скрипт завершается с кодом `1`.

### 11.2. get_params выбрасывает исключение

Если `get_params(task_id)` не находит конфигурацию для данного `task_id` и выбрасывает исключение (например, `KeyError`), оно будет перехвачено блоком `except Exception` и выведено с трассировкой. Скрипт завершится с кодом `1`.

**Решение:** убедитесь, что `parametres.py` содержит конфигурацию для вашего `task_id`.

### 11.3. Отсутствующий dataset_dir (первый запуск)

Если `params["dataset_dir"]` указывает на несуществующий каталог, скрипт считает это нормальной ситуацией и запустит этап 1. В журнале будет:

```
Каталог датасета: data/dataset_seldd_dev
  Существует: нет
  Каталог не существует.
```

### 11.4. Частичный датасет

Если `dataset_dir` существует, но внутри нет подкаталогов `metadata_dev/` или `mic_dev/`:

```
Этап 1: Генерация отражений (gpuRIR)
======================================================================
Условия выполнения:
  параметры команды изменились: False
  dataset_dir существует: True
  metadata_dev существует: False
  mic_dev существует: True

Запуск: python generate_gpuRIR_3c.py ...
```

Этап 1 будет запущен, и существующий каталог будет архивирован.

### 11.5. Датасет изменился (добавлены/удалены файлы)

Если вы вручную добавили или удалили файлы в `dataset_dir`:

```
Датасет ИЗМЕНИЛСЯ с прошлого запуска.
  предыдущий хеш: a1b2c3d4e5f6...
```

> **Внимание:** изменение датасета **не запускает** автоматически перегенерацию отражений. Этап 2 проверяет только параметры и наличие каталогов. Если вы хотите обновить данные, либо измените один из параметров `STAGE2_PARAM_KEYS`, либо удалите `dataset_dir` вручную.

Однако изменение датасета будет **зафиксировано** в хеше при сохранении состояния, и при следующем запуске будет показано, что датасет изменился.

### 11.6. Ручное удаление feat_label_dir

Если вы удалили каталог с признаками вручную:

```
Этап 2: Расчёт признаков (batch_feature_extraction)
======================================================================
Условия выполнения:
  этап 2 был выполнен: False
  параметры признаков изменились: False
  feat_label_dir существует: False
  feat_label_dir: data/feat_label_seldd_dev

Запуск: python SeldNet_3classes_80/batch_feature_extraction.py 6 ...
```

Признаки будут пересчитаны, даже если параметры не менялись.

### 11.7. Ручной сброс состояния

Если вы хотите принудительно перезапустить все этапы с нуля:

```bash
rm .maker_state/SeldNet_3classes_80_6.json
python maker.py --dir SeldNet_3classes_80 --task 6
```

Это эквивалентно первому запуску — все параметры будут считаться «новыми», и скрипт выполнит все этапы (если данные отсутствуют).

Также можно удалить весь кэш:

```bash
rm -rf .maker_state/
```

### 11.8. Ошибка на этапе 1 (генерация)

Если `generate_gpuRIR_3c.py` завершается с ненулевым кодом возврата:

```
ОШИБКА: генерация отражений (gpuRIR) завершилась с кодом возврата 1

АВАРИЙНОЕ ЗАВЕРШЕНИЕ: этап 1 завершился с ошибкой.
```

**Важные последствия:**

- Состояние **не сохраняется** (файл `.maker_state/...json` не обновляется).
- При следующем запуске скрипт снова обнаружит необходимость генерации (старые параметры будут использованы для сравнения).
- Если архивация была выполнена до генерации, старый датасет переименован (например, `dataset_dir.1/`). Его можно вернуть вручную:

```bash
mv data/dataset_seldd_dev.1 data/dataset_seldd_dev
```

### 11.9. Ошибка на этапе 2 (расчёт признаков)

Аналогично этапу 2: если `batch_feature_extraction.py` завершается с ошибкой, состояние не сохраняется, и следующий запуск повторит попытку.

### 11.10. Несколько флагов --skip одновременно

```bash
python maker.py --dir SeldNet_3classes_80 --task 6 --skip-rir --skip-feats --skip-train
```

Скрипт выполнит **только проверку параметров** и сохранит состояние. Это полезно для диагностики — посмотреть, какие параметры изменились, не запуская никаких вычислений.

### 11.11. Запуск с другим task_id при том же --dir

```bash
# Первый запуск
python maker.py --dir SeldNet_3classes_80 --task 6

# Второй запуск с другим task_id
python maker.py --dir SeldNet_3classes_80 --task 7
```

Каждый `task_id` использует **отдельный файл состояния**. Запуск с `task=7` не влияет на состояние `task=6` и наоборот. Однако оба запуска могут использовать один и тот же `dataset_dir` (если `get_params()` вернёт одинаковый путь). В этом случае:

- Если `task=6` уже сгенерировал датасет, а `task=7` использует тот же путь, этап 1 для `task=7` будет пропущен (датасет существует).
- Если `task=7` использует разные параметры генерации, но тот же `dataset_dir`, этап 1 **не будет** перезапущен (состояние `task=7` — первое выполнение, но каталог уже существует).

> **Рекомендация:** если разные `task_id` используют разные параметры генерации, убедитесь, что они указывают на разные `dataset_dir`.

### 11.12. Изменение параметров, не отслеживаемых скриптом

Если вы изменили в `parametres.py` гиперпараметры обучения (например, `learning_rate`, `batch_size`), которые **не входят** ни в `STAGE2_PARAM_KEYS`, ни в `STAGE3_PARAM_KEYS`:

```
Сравнение параметров с предыдущим запуском:
    Найдено изменённых параметров: 2
      learning_rate:
        было:  0.001
        стало: 0.0005
      batch_size:
        было:  256
        стало: 128
```

Этапы 1 и 2 будут **пропущены** (эти параметры не влияют на генерацию или признаки). Будет запущен только этап 3 с новыми гиперпараметрами.

### 11.13. Значение lowpass_hz

Параметр `lowpass_hz` берётся из `params.get("lowpass_hz", 0)`. Если в `parametres.py` ключ `lowpass_hz` отсутствует, используется значение `0` (что означает отсутствие фильтрации). Если вы хотите включить ФНЧ, добавьте этот ключ в конфигурацию:

```python
"lowpass_hz": 8000,
```

Изменение этого параметра приведёт к перезапуску этапа 2.

### 11.14. Несовместимость версий Python

Скрипт использует конструкцию `str | None` (Python 3.10+). Если вы используете Python старее 3.10, будет ошибка синтаксиса. В этом случае замените строку 197:

```python
# Было:
def compute_dataset_hash(dataset_dir: str) -> str | None:

# Стало:
def compute_dataset_hash(dataset_dir: str) -> Optional[str]:
```

и добавьте `from typing import Optional` в начало файла.

## 12. Формат файла состояния

Пример содержимого `.maker_state/SeldNet_3classes_80_6.json`:

```json
{
  "params": {
    "fs": 16000,
    "n_mics": 4,
    "label_hop_len_s": 0.1,
    "seed": 20260405,
    "max_audio_len_s": 60.0,
    "lowpass_hz": 0,
    "dataset_dir": "data/dataset_seldd_dev",
    "feat_label_dir": "data/feat_label_seldd_dev",
    "hop_len_s": 0.02,
    "nb_mel_bins": 64,
    "mel_fmin_hz": 50.0,
    "mel_fmax_hz": 8000.0,
    "use_salsalite": false,
    "raw_chunks": 300,
    "saved_chunks": 600
  },
  "dataset_hash": "a3f7b2c1d9e8f4a6b5c3d2e1f0a9b8c7",
  "last_run": "2026-04-19T14:30:22.123456",
  "stage2_executed": true,
  "stage3_executed": true
}
```

**Особенности сериализации:**

- Значения `float("inf")` сохраняются как строка `"__inf__"` и восстанавливаются при чтении.
- Объекты `Path` преобразуются в строки.
- Все остальные типы (int, float, str, bool, list, dict) сохраняются как есть.

## 13. Устранение неполадок

| Проблема                                                                                  | Возможная причина                                                                                     | Решение                                                                                                                                                                    |
| ------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Скрипт не находит `parametres.py`                                                | Неправильное значение `--dir` или запуск не из корневого каталога | Убедитесь, что вы запускаете `maker.py` из корня проекта и `--dir` указывает на существующий каталог     |
| `KeyError` при вызове `get_params`                                                   | Нет конфигурации для данного `task_id`                                                     | Проверьте `parametres.py` и добавьте конфигурацию для нужного `task_id`                                                               |
| Этап 1 не запускается, хотя нужен                                       | `dataset_dir`, `metadata_dev/`, `mic_dev/` существуют, а параметры не менялись    | Удалите `dataset_dir` вручную или измените один из `STAGE2_PARAM_KEYS`                                                                         |
| Этап 2 не запускается, хотя нужны новые признаки           | Указан `--skip-feats` или `feat_label_dir` существует                                          | Уберите `--skip-feats` или удалите `feat_label_dir`                                                                                                          |
| После ошибки этапа 2 состояние не обновляется               | Это ожидаемое поведение: при ошибке состояние не сохраняется      | Исправьте ошибку и перезапустите. Если архивация была выполнена, верните старый каталог вручную |
| Архивный каталог занял много места                                  | Многократные запуски с изменением параметров создают копии        | Регулярно удаляйте старые архивы (`dataset_dir.1/`, `dataset_dir.2/` и т.д.)                                                                  |
| Лог-файл пуст                                                                          | Файл был создан, но скрипт завершился до вывода                                | Проверьте, что директория для лога доступна для записи                                                                              |
| Повторный запуск в тот же день перезаписывает модель | Имя модели включает только дату, не время                                           | При необходимости вручную переименуйте файлы модели после первого запуска                                        |
